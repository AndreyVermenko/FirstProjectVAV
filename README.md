# 🚀 Космолёт

Браузерная аркада на чистом JavaScript: **вид из кабины космолёта**, полёт в открытом
космосе и уклонение от встречных астероидов. Управление — стрелками на клавиатуре.

![Геймплей](docs/screenshots/gameplay.png?raw=true)

## 🎮 Как играть

1. Откройте игру (см. [«Запуск»](#-запуск)).
2. Нажмите **НАЧАТЬ ПОЛЁТ** или **Пробел**.
3. Уклоняйтесь от астероидов, которые летят прямо на кабину.
4. Каждое столкновение отнимает единицу прочности корпуса (их три). На нуле — конец полёта.
5. Цель — пролететь как можно большую дистанцию. Рекорд сохраняется в браузере.

### Управление

| Клавиша | Действие |
|---|---|
| **←** / **A** | сместить кабину влево |
| **→** / **D** | сместить кабину вправо |
| **↑** / **W** | сместить кабину вверх |
| **↓** / **S** | сместить кабину вниз |
| **Пробел** / **Enter** | старт и рестарт |
| **M** | вкл/выкл звук |

## 🛰️ Что реализовано (по задаче)

Игра закрывает все пункты исходного задания:

1. ✅ **Вид изнутри кабины космолёта** — остекление, корпус и приборная панель с показаниями.
2. ✅ **Полёт в открытом космосе** — звёздное поле проносится навстречу, создавая ощущение скорости.
3. ✅ **Астероиды навстречу — нужно уклоняться** — камни приближаются из глубины и растут; нужно увести кабину в сторону.
4. ✅ **Управление стрелками** — вправо/влево/вверх/вниз (и дублирующие WASD).
5. ✅ **Игра в браузере на JavaScript** — HTML5 Canvas, без зависимостей и сборки.

## ▶️ Запуск

**Способ 1 — просто открыть файл.** Игра не требует сервера: откройте `index.html`
двойным щелчком в любом современном браузере.

**Способ 2 — локальный сервер** (нужен [Node.js](https://nodejs.org/)):

```bash
npm start
# затем откройте http://localhost:8000/
```

## 🧪 Тесты

Игровая логика (проекция, столкновения, появление астероидов, кривая сложности)
вынесена в отдельный модуль и покрыта unit-тестами на встроенном тест-раннере Node
(без внешних зависимостей):

```bash
npm test
```

Тесты автоматически прогоняются в GitHub Actions при каждом пуше и pull request
(см. [`.github/workflows/ci.yml`](.github/workflows/ci.yml)).

## 🗂️ Структура проекта

```
.
├── index.html           # страница игры
├── css/style.css        # оформление: тема космоса, экраны меню
├── js/
│   ├── engine.js        # чистая логика мира (проекция, столкновения) — тестируется
│   └── game.js          # рендер на Canvas, игровой цикл, ввод, состояния
├── tests/engine.test.js # unit-тесты ядра
├── serve.js             # крошечный статический сервер для `npm start`
├── favicon.svg          # иконка
└── docs/screenshots/    # скриншоты
```

Логика отделена от отрисовки намеренно: `engine.js` не зависит от DOM и Canvas,
поэтому её можно проверять в Node, а `game.js` отвечает только за визуализацию и ввод.
